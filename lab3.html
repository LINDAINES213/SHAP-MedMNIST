Dataset: The PathMNIST is based on a prior study for predicting survival from colorectal cancer histology slides, providing a dataset (NCT-CRC-HE-100K) of 100,000 non-overlapping image patches from hematoxylin & eosin stained histological images, and a test dataset (CRC-VAL-HE-7K) of 7,180 image patches from a different clinical center. The dataset is comprised of 9 types of tissues, resulting in a multi-class classification task. We resize the source images of 3×224×224 into 3×28×28, and split NCT-CRC-HE-100K into training and validation set with a ratio of 9:1. The CRC-VAL-HE-7K is treated as the test set.

# ========= 4) Define Simple CNN Model (already provided) =========
import torch.nn as nn
import torch.nn.functional as F

class SimpleCNN(nn.Module):
    def __init__(self, n_channels, n_classes):
        super(SimpleCNN, self).__init__()
        self.conv1 = nn.Conv2d(n_channels, 16, kernel_size=3, padding=1)
        self.pool = nn.MaxPool2d(2, 2)
        self.fc1 = nn.Linear(16*14*14, n_classes)

    def forward(self, x):
        x = self.pool(F.relu(self.conv1(x)))
        x = x.reshape(x.size(0), -1)
        x = self.fc1(x)
        return x

model = SimpleCNN(info['n_channels'], n_classes)
device = torch.device("cuda" if torch.cuda.is_available() else "cpu")
model.to(device)

print("Model class defined and ready.")

# HINT: x will come as a numpy array with shape (N, H, W, C).
#       Convert it to a torch tensor (N, C, H, W), run through the model,
#       return probabilities as numpy.
def model_forward(x: np.ndarray) -> np.ndarray:
    model.eval()
    with torch.no_grad():
        if isinstance(x, list):
            x = np.stack(x, axis=0)
        x = x.astype(np.float32)

        # Si viene en [0,255], escalar a [0,1]
        if x.max() > 1.0:
            x = x / 255.0

        # Asegurar dimensión batch
        if x.ndim == 3:
            x = x[None, ...]  # (1,H,W,C)

        # Alinear canales esperados vs. provistos
        if x.shape[-1] != info['n_channels']:
            if x.shape[-1] == 1 and info['n_channels'] == 3:
                x = np.repeat(x, 3, axis=-1)  # gris -> RGB
            elif x.shape[-1] == 3 and info['n_channels'] == 1:
                x = x[..., :1]                # RGB -> gris
            else:
                raise ValueError(f"Se esperaban {info['n_channels']} canales, llegó {x.shape[-1]}.")

        # HWC -> CHW y normalización (mean=0.5, std=0.5)
        x_t = torch.from_numpy(x).permute(0, 3, 1, 2)
        x_t = (x_t - 0.5) / 0.5
        x_t = x_t.to(device)

        logits = model(x_t)
        probs = F.softmax(logits, dim=1).cpu().numpy()  # <-- línea correcta
    return probs

# ===== Variante B: panel multiclase por imagen + accuracy =====
num_imgs = 5
idxs = np.random.choice(len(test_dataset), size=num_imgs, replace=False)

correct = 0

for k, idx in enumerate(idxs, start=1):
    img_t, true_idx = test_dataset[idx]
    img_np = img_t.permute(1, 2, 0).cpu().numpy().astype(np.float32)
    img_np = np.clip((img_np * 0.5) + 0.5, 0, 1)

    # Predicción
    probs = model_forward(img_np[None, ...])[0]
    pred_idx = int(probs.argmax())
    pred_prob = float(probs[pred_idx])
    is_correct = (pred_idx == int(true_idx))
    correct += int(is_correct)

    # SHAP
    masker = shap.maskers.Image("blur(28,28)", img_np.shape)
    explainer = shap.Explainer(model_forward, masker, output_names=[id2label[i] for i in range(n_classes)])
    explanation = explainer(img_np[None, ...])

    vals = explanation.values
    # Construimos la lista por clase si es 5D; si es 4D, no hay desagregación por clase
    if vals.ndim == 5:
        shap_values_list = [vals[..., j] for j in range(n_classes)]  # lista de (1,H,W,C) por clase
        shap.image_plot(shap_values_list, img_np[None, ...], show=False)
    elif vals.ndim == 4:
        shap.image_plot(vals, img_np[None, ...], show=False)
    else:
        raise ValueError(f"Dimensión inesperada en explanation.values: {vals.shape}")

    # Añadimos un título global informativo a la figura recién generada por SHAP
    fig = plt.gcf()
    check = "✔" if is_correct else "✘"
    fig.suptitle(
        f"Imagen {k}/{num_imgs} — Real: {id2label[int(true_idx)]} | "
        f"Pred: {id2label[pred_idx]} ({pred_prob:.0%}) {check}",
        fontsize=12
    )
    plt.show()

# Accuracy global al final
acc = correct / num_imgs
print(f"Aciertos: {correct}/{num_imgs} — Accuracy: {acc:.2%}")

========= 8) Reflexion =========¶

Al analizar los resultados de las cinco imágenes, se observa que el modelo toma sus decisiones principalmente en función de patrones de color y textura. En los casos de mucus, por ejemplo, las regiones violetas y densas parecen ser determinantes para la predicción. Lo mismo ocurre con cancer-associated stroma, donde las fibras de color rosa fueron son destacadas por los mapas SHAP. En contraste, las imágenes que cuentan con un mismo patron en toda la imagen son clasificadas como background, lo cual tiene sentido porque no muestran ningún tipo de estructura clara. Sin embargo, en los errores como en la confusión de adipose se confundió con background la atención no funciona correctamente. El mapa no señala estructuras definidas, lo que refleja que el modelo estaba inseguro y se apoyó en patrones débiles o superficiales. Este contraste muestra que los heatmaps no solo explican, sino que también permiten diagnosticar las debilidades del modelo. En términos de utilidad, la interpretabilidad es valiosa porque no se trata solo de justificar predicciones, sino de guiar mejoras. Ver cómo el modelo confunde adipose con background indica que hace falta enriquecer el dataset con ejemplos más variados de adiposo, o bien aplicar técnicas de preprocesamiento y aumentación que resalten sus características diferenciales. De esta manera, la explicabilidad se convierte en una herramienta práctica para mejorar el desempeño y la confiabilidad del modelo.
